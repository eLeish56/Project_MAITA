📋 RINGKASAN EKSEKUTIF
─────────────────────────────────────────────────────────────────────────────

Sistem POS Terintegrasi dengan Marketplace & Procurement adalah platform 
komprehensif yang mengelola:

  • Penjualan Online (Marketplace)
  • Penjualan Offline (POS Counter)
  • Manajemen Inventory (Stok & Batch Tracking)
  • Workflow Procurement (PR → PO → GR → Invoice)
  • Reporting & Analytics
  • User Management (Role-based)

Total: 6 Aktor, 50+ Use Cases, 100+ Fitur, 6 Subsystem


═══════════════════════════════════════════════════════════════════════════════
1️⃣  AKTOR: CUSTOMER (PELANGGAN MARKETPLACE)
═══════════════════════════════════════════════════════════════════════════════

DESKRIPSI:
  Pengguna publik yang melakukan self-register dan berbelanja melalui 
  marketplace online.

JUMLAH FITUR: 16
PRIORITY: HIGH
STATUS: ✅ Fully Implemented

FITUR-FITUR:
─────────────────────────────────────────────────────────────────────────────

▸ AUTHENTICATION (3 fitur)
  1. Register (Daftar akun baru)
  2. Login (Login dengan email/username)
  3. Logout (Keluar dari sistem)

▸ MARKETPLACE SHOPPING (7 fitur)
  4. Browse Products (Lihat daftar produk)
  5. Search/Filter Products (Cari & filter produk)
  6. View Product Details (Lihat detail produk)
  7. Add to Cart (Tambah ke keranjang)
  8. View Cart (Lihat isi keranjang)
  9. Update Cart Quantity (Ubah jumlah item)
  10. Remove from Cart (Hapus item dari keranjang)

▸ CART & CHECKOUT (3 fitur)
  11. Clear Cart (Kosongkan keranjang)
  12. Checkout (Lanjut ke pembayaran)
  13. Process Payment (Lakukan pembayaran online)

▸ ORDER MANAGEMENT (3 fitur)
  14. View Orders (Lihat pesanan saya)
  15. View Order Details (Lihat detail pesanan)
  16. Track Order Status (Lacak status pesanan)

PLUS: Edit Profile, View Profile

5️⃣  AKTOR: SUPPLIER (VENDOR / PENYEDIA)
═══════════════════════════════════════════════════════════════════════════════

DESKRIPSI:
  Pihak eksternal (vendor) yang menyediakan produk. Tidak memiliki akses 
  login ke sistem, tetapi berinteraksi melalui dokumen PO, delivery, dan 
  invoice.

JUMLAH FITUR: 5
PRIORITY: LOW-MEDIUM
STATUS: 📋 Partial (Receiving side only)

FITUR-FITUR (DALAM KONTEKS SISTEM):
─────────────────────────────────────────────────────────────────────────────

▸ PRODUCT CATALOG
  1. Provide Product List (Data produk tersimpan di sistem)
  2. Maintain Product Information (Harga, spesifikasi)

▸ PURCHASE ORDER PROCESSING
  3. Receive Purchase Order (Terima PO dari sistem)
  4. Deliver Goods (Kirim barang)

▸ INVOICING
  5. Submit Invoice (Kirim invoice untuk pembayaran)

Note: Supplier tidak login langsung, tapi berinteraksi melalui:
  • Email/dokumen fisik untuk PO
  • Physical delivery untuk barang
  • Email/dokumen fisik untuk invoice

🔄 WORKFLOW UTAMA
═══════════════════════════════════════════════════════════════════════════════

WORKFLOW 1: MARKETPLACE SHOPPING
─────────────────────────────────────────────────────────────────────────────
Customer Browse → Search → View Details → Add to Cart → Checkout → 
Payment → Order Placed → View Status → Cashier Process → Completed

WORKFLOW 2: POS COUNTER TRANSACTION
─────────────────────────────────────────────────────────────────────────────
Cashier New Transaction → Scan Items → Check Stock → Update Qty → 
Complete → Select Payment → Print Receipt → Done

WORKFLOW 3: COMPLETE PROCUREMENT
─────────────────────────────────────────────────────────────────────────────
Supervisor Create PR → Approve PR → Create PO → Send to Supplier → 
Receive Goods (GR) → Record Inventory → Receive Invoice → Process Payment → 
Complete

WORKFLOW 4: STOCK REORDER VIA ANALYSIS
─────────────────────────────────────────────────────────────────────────────
Supervisor View Analysis → Identify Need → Create PR for Fast-Moving Items → 
Approve & Create PO → Monitor Delivery → Receive & Record → Complete

📞 NOTES PENTING
═══════════════════════════════════════════════════════════════════════════════

1. Supervisor adalah role PALING POWERFUL dengan 60+ fitur
2. Procurement workflow PANJANG: PR → PO → GR → Invoice (4 dokumen)
3. Stock management REAL-TIME: dicek saat add to cart
4. Multi-channel sales: POS (offline) + Marketplace (online)
5. Role-based access via Middleware (IsAdmin, IsSupervisor, etc)
6. Payment methods CONFIGURABLE per toko
7. Batch tracking mandatory untuk produk dengan expiry
8. Semua transaksi AUDITABLE dengan user & timestamp
9. Stock hanya BERKURANG setelah payment success
10. Invoice dapat di-mark as PAID setelah pembayaran confirmed
